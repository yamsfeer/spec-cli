# AI 提示词模板

> 以下是与 AI 编码助手协作的常用提示词模板，根据实际项目调整。

## 创建新功能

```
我需要实现一个 [功能名称] 功能。请参考以下信息：

**需求背景**：
[简要描述业务需求]

**技术约束**：
- 使用 [技术栈]
- 遵循已有的 [代码风格/命名规范]
- 需要兼容现有的 [模块/接口]

**期望产出**：
1. [文件路径]：[功能代码]
2. [文件路径]：[测试代码]
3. 如有需要，更新相关的类型定义

**验收标准**：
- [标准 1]
- [标准 2]

在开始之前，如果有不确定的地方请先和我确认。
```

## 修复 Bug

```
我需要修复一个 Bug：

**问题描述**：
[详细描述 Bug 现象、复现步骤]

**期望行为**：
[描述正确的行为应该是什么]

**相关文件**：
- [文件路径]
- [文件路径]

**日志/错误信息**：
```
[粘贴相关错误日志]
```

请分析根因并给出修复方案。
```

## 代码审查

```
请审查以下代码片段/文件的代码质量，关注：

1. **正确性**：逻辑是否正确，是否有边界情况遗漏
2. **安全性**：是否有安全漏洞（注入、权限绕过等）
3. **性能**：是否有性能瓶颈
4. **可读性**：代码是否清晰易懂
5. **规范性**：是否符合项目编码规范

```typescript
[粘贴代码]
```
```

## 编写测试

```
请为以下函数/组件编写单元测试：

**被测代码**：
```typescript
[粘贴代码]
```

**测试要求**：
- 覆盖率目标：≥80%
- 需要覆盖的异常场景：[列出特定的异常情况]
- 测试框架：[Vitest / Jest]

请提供完整的测试代码。
```

## 重构代码

```
请重构以下代码：

**重构目标**：
- [可读性/性能/复用性/解耦]

**原始代码**：
```typescript
[粘贴代码]
```

**约束**：
- 不要改变对外的 API 签名
- 保持向后兼容
- 现有的测试必须全部通过
```
