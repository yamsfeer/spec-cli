# 认证页面流

## 用户注册流程

```mermaid
flowchart TD
    A[首页] -->|点击注册| B[注册页面]
    B --> C{填写注册信息}
    C --> D[提交注册表单]
    D -->|参数校验失败| E[显示错误提示]
    E --> C
    D -->|校验通过| F[调用 POST /auth/register]
    F -->|邮箱已注册| G[提示"邮箱已被注册"]
    G --> C
    F -->|注册成功| H[自动登录]
    H --> I[跳转到首页/控制台]
```

## 用户登录流程

```mermaid
flowchart TD
    A[首页] -->|点击登录| B[登录页面]
    B --> C{选择登录方式}
    C -->|邮箱密码| D[填写邮箱/密码]
    D --> E[提交登录表单]
    E -->|参数校验失败| F[显示错误提示]
    F --> D
    E -->|调用 POST /auth/login| G{验证结果}
    G -->|账号不存在| H[提示"账号不存在"]
    G -->|密码错误| I[提示"密码错误"]
    G -->|账号锁定| J[提示"账号已锁定，请X分钟后重试"]
    G -->|登录成功| K[保存 Token]
    K --> L[跳转到目标页面]
    H --> D
    I --> D
```

## 找回密码流程

```mermaid
flowchart TD
    A[登录页] -->|忘记密码| B[找回密码页]
    B --> C[输入注册邮箱]
    C --> D[发送重置链接]
    D -->|邮箱未注册| E[提示"该邮箱未注册"]
    D -->|发送成功| F[提示"重置链接已发送"]
    F --> G[用户查收邮件]
    G --> H[点击邮件中的重置链接]
    H --> I[设置新密码页]
    I --> J[输入新密码并确认]
    J -->|两次不一致| K[提示"密码不一致"]
    J -->|格式无效| L[提示密码要求]
    J -->|重置成功| M[提示"密码已重置"]
    M --> N[跳转到登录页]
    K --> J
    L --> J
```

## 页面流转说明

| 页面 | 可访问状态 | 进入方式 | 离开方式 |
|------|-----------|---------|---------|
| 首页 | 所有用户 | 直接访问 | 导航链接 |
| 登录页 | 未登录 | 点击登录 / Token 过期 | 登录成功 / 点击返回 |
| 注册页 | 未登录 | 点击注册 | 注册成功 / 点击返回登录 |
| 找回密码页 | 未登录 | 点击忘记密码 | 发送邮件 / 取消 |
| 重置密码页 | 未登录 | 邮件链接（含 token） | 重置成功 / token 过期 |
| 控制台 | 已登录 | 登录成功后 | 退出登录 / 导航 |
