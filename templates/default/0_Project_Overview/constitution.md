## 通用原则参考

以下是一些可参考的通用软件工程原则（请根据项目实际情况调整）：

1. **文档即源码**：Spec 文档与代码同等重要，必须可校验、可追溯
2. **简单优先**：选择最简单的方案，除非复杂度能带来明确的收益
3. **约定优于配置**：确定清晰的默认约定，减少配置成本
4. **引用优先于复制**：同一信息只在一处定义，其他地方通过引用关联
5. **先设计，再编码**：设计文档通过评审后方可进入编码阶段

## 决策流程

当遇到影响多个模块的重大决策时，应遵循以下流程：

1. 识别问题：明确需要做什么决策
2. 调研方案：列出至少 2 个可行方案
3. 评估权衡：分析各方案的优缺点
4. 做出决策：选定方案并记录理由
5. 记录 ADR：在 `7_Changelog/adr.md` 中创建架构决策记录
